## 1. Migrations (m0017 — aditivas)
```sql
CREATE TABLE IF NOT EXISTS asset_questions (
  id           INTEGER PRIMARY KEY AUTOINCREMENT,
  diagram_type TEXT NOT NULL,     -- 'acoes' | 'imobiliario'
  criterio     TEXT,              -- 'ROE','CAGR'…
  pergunta     TEXT NOT NULL,
  peso         REAL NOT NULL DEFAULT 1,
  sort_order   INTEGER NOT NULL DEFAULT 0,
  ativo        INTEGER NOT NULL DEFAULT 1
);
CREATE TABLE IF NOT EXISTS asset_answers (
  asset_id    INTEGER NOT NULL REFERENCES portfolio_assets(id) ON DELETE CASCADE,
  question_id INTEGER NOT NULL REFERENCES asset_questions(id) ON DELETE CASCADE,
  resposta    INTEGER NOT NULL DEFAULT 0,   -- 1=sim, 0=não
  PRIMARY KEY (asset_id, question_id)
);
```
**Mapa classe → diagram_type:** `acoes_nac`, `acoes_int` → `acoes`; `fii`, `reit` → `imobiliario`. `rf`/`rf_int`/
`cripto` → **sem score** (não recebem perguntas; nota neutra/n.a. — ver §4 e F4.4).

**Seeds padrão (botões "Restaurar padrões"/"Usar modelo"):**
- *acoes:* ROE histórico > 8%? · CAGR receita/lucro > 5% em 5 anos? · histórico de dividendos? · investe em P&D?
  (setor obsoleto = sempre não) · > 30 anos de mercado?
- *imobiliario:* é líder do setor? · setor com > 100 anos? · boa governança? (corrupção = sempre não) · livre de
  controle estatal/cliente único? · Dív.Líq/EBITDA < 3 em 5 anos?

## 2. Nota (normalizada — overview §4)
Para o diagrama da classe do ativo: `peso_total = Σ peso` (perguntas `ativo=1`);
`positivos = Σ peso(resposta=1)`, `negativos = Σ peso(resposta=0)`; `bruta = positivos − negativos`;
**`nota = bruta / peso_total × 10`** (∈ −10..+10). Com 10 perguntas peso 1 → começa em −10, +2 por "Sim"
(reproduz o print). Funciona com **qualquer quantidade** de perguntas (peso aplicado conforme o total).
Exposições: `pontos_positivos`, `pontos_negativos`, `nota`. **Só `nota > 0` recebe aporte** (F4.4).

> Repo `score_repo.compute_nota(db, asset_id)` (puro, testável). Perguntas **não respondidas** contam como
> "não" (0) — coerente com o print (toggle desligado = não).

## 3. Endpoints (router `portfolio.py`)
- **`GET /api/investments/questions?diagram_type=acoes|imobiliario`** → lista ordenada.
- **`POST /api/investments/questions`** `{ diagram_type, criterio?, pergunta, peso?, sort_order? }`.
- **`PATCH /api/investments/questions/{id}`** · **`DELETE /api/investments/questions/{id}`**.
- **`POST /api/investments/questions/restore-defaults?diagram_type=`** → repõe os seeds.
- **`GET /api/investments/assets/{id}/answers`** → `{ answers:{question_id:bool}, pontos_positivos, pontos_negativos, nota }`.
- **`PUT /api/investments/assets/{id}/answers`** `{ [question_id]: bool }` → grava respostas e devolve a nota recalculada.
- Editar/remover pergunta **recalcula a nota** de todos os ativos do diagrama (ou recalcula sob demanda no GET).
- `GET /api/investments` (F4.1) passa a incluir `nota` por ativo (coluna "Nota" da lista).

## 5. Edge cases
- 0 perguntas no diagrama → `peso_total=0` → nota indefinida (tratar como 0/n.a.; não quebra a divisão).
- Adicionar/remover pergunta muda `peso_total` → todas as notas do diagrama mudam (recalcular).
- Classe sem diagrama (rf/cripto): sem perguntas; nota n.a. — na F4.4 essas classes seguem o alvo de classe sem
  ponderação por nota (distribuir igualmente ou por valor; ver F4.4 §edge).
