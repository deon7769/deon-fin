## 1. Contexto & objetivo

O app roda em produção na VPS (`/opt/projetos/financas-agent`) como **um único** serviço Docker Compose
`financas-agent`, atrás do **Traefik** em `https://fin.deonlab.tech`, com middleware `tailscale-only@docker`
(acesso privado), TLS Let's Encrypt e SQLite em volume (`./data:/app/data`). O `Dockerfile` builda só o
Python e roda `python -m src.cli serve --host 0.0.0.0 --port 8000`. O deploy é `scripts/vps_deploy.sh`
(backup do `data/financas.db` → `pytest` → `docker compose build` → `up -d` → smoke em `/api/health` →
logs). O watchdog/healthcheck do host monitoram o container **pelo nome** `financas-agent`
(`/opt/infra-healthcheck.sh`, `/opt/infra-watchdog.sh`, `infra-watchdog.timer`).

O front legado (Jinja "Raio-X Financeiro") é servido em `/` pela própria FastAPI. Agora há um front **Next**
(`web/`, F0.2). O objetivo é servi-lo **na mesma origem** da API (`/api`) — sem CORS em produção, com o
Basic Auth existente cobrindo tudo — **preservando** o modelo de 1 container / 1 router do Traefik e o
fluxo de deploy/observabilidade atuais.

## 2. Decisão de arquitetura

### Abordagem escolhida (A): Next **estático** embutido na imagem, servido pela FastAPI
- O dashboard é **client-rendered** (TanStack Query busca de `/api`) — não precisa de SSR. Logo, build do
  Next com `output: 'export'` gera HTML/JS/CSS estáticos (`web/out`). A **FastAPI serve** esses arquivos.
- **Mesma origem:** o front chama `/api` (caminho relativo). Sem CORS em produção. O Basic Auth (middleware
  já existente) cobre HTML, assets e API de uma vez (privado via Tailscale).
- **Infra praticamente intacta:** continua 1 container, 1 router Traefik, porta 8000, mesmo nome
  `financas-agent` (watchdog/healthcheck seguem válidos), mesmo volume SQLite. O `Dockerfile` ganha um
  **estágio Node** de build; nada novo no `docker-compose.yml`/Traefik.

### Alternativa (B), documentada e NÃO escolhida agora: container Node (Next standalone) + Traefik path-routing
- Dois serviços: `financas-api` (FastAPI) e `financas-web` (Next `output:'standalone'`), com o Traefik
  roteando `PathPrefix(/api)` → API e o resto → Next. Suporta SSR/ISR.
- **Custo:** segundo runtime/imagem, segundo router/healthcheck/watchdog, mais superfície de auth (Basic
  Auth teria de cobrir os dois). Só vale se, no futuro, precisarmos de SSR/edge. Manter como caminho de
  evolução, não para agora.

## 3. Mudanças

### 3.1 `next.config` (em `web/`, F0.2 deixou os ganchos)
```js
// web/next.config.mjs
const isProd = process.env.NODE_ENV === 'production';
export default {
  output: 'export',                 // gera web/out (estático)
  trailingSlash: true,              // index.html por rota → casa com file-serving
  images: { unoptimized: true },    // export não usa o otimizador
  // basePath/assetPrefix permanecem vazios: servimos na raiz same-origin.
};
```
- `web/lib/api.ts` (F0.2): `baseURL = process.env.NEXT_PUBLIC_API_URL ?? '/api'`. **Produção:** `/api`
  (relativo, same-origin). **Dev:** `.env.local` com `NEXT_PUBLIC_API_URL=http://127.0.0.1:8000/api`
  (cross-origin, CORS de F0.1 liberando `localhost:3000`).
- Rotas client-side: como é export, garantir navegação por `next/link`; o file-serving + fallback (§3.3)
  resolve refresh em rotas profundas.

### 3.2 `Dockerfile` — multi-stage (Node build → imagem Python)
```dockerfile
# ---------- estágio 1: build do front ----------
FROM node:20-slim AS web
WORKDIR /web
COPY web/package*.json ./
RUN npm ci
COPY web/ ./
ENV NEXT_PUBLIC_API_URL=/api
RUN npm run build           # next build (output:'export') → /web/out

# ---------- estágio 2: runtime Python (como hoje) ----------
FROM python:3.12-slim-bookworm
WORKDIR /app
ENV PYTHONDONTWRITEBYTECODE=1 PYTHONUNBUFFERED=1
RUN apt-get update && apt-get install -y --no-install-recommends ca-certificates \
    && rm -rf /var/lib/apt/lists/*
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY src ./src
COPY scripts ./scripts
COPY --from=web /web/out ./web_dist     # build estático do Next
RUN mkdir -p /app/data
EXPOSE 8000
CMD ["python", "-m", "src.cli", "serve", "--host", "0.0.0.0", "--port", "8000"]
```
- `docker compose build` (já no `vps_deploy.sh`) passa a buildar os **dois** estágios. **Node não é
  necessário na VPS** — tudo acontece no build da imagem.

### 3.3 FastAPI serve o SPA (em `src/web/app.py`)
Servir `web_dist` na raiz, **sem** colidir com `/api/*` nem com o `/static` do legado:
```python
from fastapi.staticfiles import StaticFiles
from fastapi.responses import FileResponse
WEB_DIST = Path(os.environ.get("WEB_DIST_DIR", "web_dist"))

# Assets do Next (hashados) — alta prioridade
if (WEB_DIST / "_next").exists():
    app.mount("/_next", StaticFiles(directory=str(WEB_DIST / "_next")), name="next-assets")

# Fallback SPA: qualquer rota que não seja /api, /static, /_next → index.html do Next
@app.get("/{full_path:path}", include_in_schema=False)
def spa(full_path: str):
    if full_path.startswith(("api", "static", "_next")):
        raise HTTPException(404)
    candidate = WEB_DIST / full_path
    if candidate.is_file():
        return FileResponse(candidate)
    index = WEB_DIST / "index.html"
    if index.exists():
        return FileResponse(index)
    raise HTTPException(404)
```
- **Precedência:** os routers `/api/*` e o mount `/static` (legado) são registrados **antes** do catch-all,
  então têm prioridade. `/api/health` permanece público (fora do Basic Auth) — middleware inalterado.
- **Legado:** mover a rota Jinja `/` para `/legacy` (mantendo `/static`), e o Next assume `/`. Toggle de
  rollback: env `LEGACY_UI=1` faz `/` voltar a renderizar o legado (sem rebuild). Documentar no `.env.example`.
- **Auth:** o middleware Basic Auth (`APP_PASSWORD`) já embrulha todas as rotas → cobre HTML/assets/API.
  Em dev (`APP_PASSWORD` vazio) fica desligado (F0.1). O browser pede credenciais uma vez e as reusa no
  `fetch('/api/...')` same-origin.

### 3.4 `docker-compose.yml` / Traefik
- **Sem mudanças necessárias.** Continua 1 serviço, porta 8000, `Host(fin.deonlab.tech)`, `tailscale-only`,
  TLS. (Opcional: header de cache para `/_next/*` via middleware Traefik — não obrigatório.)

### 3.5 `scripts/vps_deploy.sh`
- Mantém backup do SQLite, `pytest`, `docker compose build` (agora builda o front também), `up -d`.
- **Adicionar smoke do front** ao lado do `/api/health`: requisitar `/` (com credenciais Basic Auth de
  teste, ou via Tailscale) e checar status 200 + um marcador do HTML do Next (ex.: `id="__next"` ou um
  `<meta name="app" content="deon-fin">`). Falha → aborta antes do `up` final/loga.
- Continua válido o cuidado de isolamento de teste (pytest não herda `APP_PASSWORD` de produção —
  vps-foundation-design §Problem).

### 3.6 CI opcional (GitHub Actions)
- Workflow em push/PR: `pip install -r requirements.txt && pytest -q`; `cd web && npm ci && npm run build`
  (e `npm run lint`). Pega quebra de build/tests **antes** do deploy na VPS. (A VPS segue como fonte
  operacional; o CI é rede de segurança, não substitui o `vps_deploy.sh`.)

## 4. Regras / edge cases
- **Refresh em rota profunda** (`/transacoes`): o fallback SPA devolve `index.html`; o roteador do Next
  resolve no cliente. `trailingSlash:true` ajuda o file-serving direto quando o arquivo existe.
- **`/api` 404 vs SPA:** o catch-all exclui `api/`, então chamadas de API erradas retornam JSON de erro
  (F0.1), não o HTML do SPA.
- **Cache de assets:** `_next/*` é hashado → cache longo seguro; `index.html` deve ter `Cache-Control:
  no-cache` para o deploy refletir na hora.
- **Rollback:** `LEGACY_UI=1` (instantâneo) ou `git revert` + `vps_deploy.sh` (com backup do DB já feito).
- **Watchdog/healthcheck:** nome do container inalterado → cobertura mantida sem editar
  `/opt/infra-*.sh`.
- **Tamanho da imagem:** multi-stage descarta o Node; só o `web_dist` estático entra na imagem final.
- **Migrations no deploy:** rodam no `Database.__init__` (F0.1, idempotentes) ao subir o container; o
  backup do SQLite pelo script protege contra regressão.

## 6. Critérios de aceite
- `docker compose build` produz uma imagem que serve o Next em `/` e a API em `/api`, na **mesma origem**,
  sem CORS em produção e atrás do Basic Auth.
- `https://fin.deonlab.tech/` carrega o dashboard Next; refresh em rotas profundas funciona; chamadas
  `/api/*` respondem; `/api/health` segue público.
- Traefik/Tailscale/TLS/volume SQLite e o nome do container permanecem; watchdog/healthcheck continuam
  cobrindo `financas-agent`.
- `vps_deploy.sh` faz backup, roda testes, builda (front+back), sobe, e o smoke do front + `/api/health`
  passam; rollback via `LEGACY_UI=1` funciona.
- Front legado acessível em `/legacy` durante a transição.
