## 1. Objetivo

Criar a tela nova do simulador financeiro usando o layout do app Next. A primeira versão deve priorizar
cenários úteis e verificáveis, sem substituir a regra de negócio já existente em `src/agent/simulator.py`.

## 2. Escopo funcional

- Criar rota `web/app/(app)/simulador/page.tsx` e item no menu principal.
- Consumir os endpoints existentes:
  - `/api/simular` para cenários financeiros gerais;
  - `/api/amortizacao` para projeções de amortização.
- Expor dois modos por abas ou controle segmentado:
  - **Cenários:** entradas principais do simulador, cards de resultado e comparação.
  - **Amortização:** saldo, taxa, parcela/prazo e tabela/resumo de evolução.
- Validar entrada no frontend antes de chamar a API, com mensagens curtas e campos numéricos seguros.
- Mostrar estados de loading, erro, resultado vazio e resultado calculado.

## 3. Arquitetura esperada

- Backend: manter a lógica em `src/agent/simulator.py`; criar wrappers novos apenas se for necessário estabilizar o contrato da UI.
- Frontend:
  - `web/hooks/useSimulator.ts`
  - `web/lib/simulator.ts` para formatação e normalização de payloads
  - `web/components/simulador/*`
  - testes em `web/tests/simulator.test.ts`
- Visualização: cards de resumo para resultados principais e tabela compacta para amortização. Gráficos só entram se o endpoint fornecer série temporal suficiente.

## 4. Critérios de aceite

- `/simulador` renderiza no Next e chama somente `/api/*` relativo.
- Simulação e amortização podem ser executadas sem recarregar a página.
- Inputs inválidos não disparam requests e mostram feedback claro.
- Resultado do frontend bate com respostas dos endpoints backend para fixtures conhecidas.
- A tela continua funcional após refresh em rota profunda no deploy same-origin.

## 5. Testes mínimos

- Backend: regressão dos endpoints `/api/simular` e `/api/amortizacao` com payloads representativos.
- Frontend: validação de formulário, chamada mockada da API, renderização de cards e tabela.
- Build: `npm test -- --run`, `npm run typecheck`, `npm run lint`, `npm run build`.
