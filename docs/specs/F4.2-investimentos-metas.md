## 1. Migrations (m0016 — aditivas)
```sql
CREATE TABLE IF NOT EXISTS allocation_targets (
  asset_class TEXT PRIMARY KEY,   -- 'acoes_nac','acoes_int','etf','fii','reit','cripto','rf','rf_int'
  target_pct  REAL NOT NULL DEFAULT 0
);
CREATE TABLE IF NOT EXISTS investment_profile (
  id            INTEGER PRIMARY KEY CHECK (id=1),
  perfil        TEXT,            -- 'conservador'|'moderado'|'arrojado'|'custom'
  ultimo_aporte REAL,            -- usado pela F4.4 (UX "último aporte")
  updated_at    TEXT DEFAULT (datetime('now'))
);
```
**Seed:** as 8 classes em `allocation_targets` com `0%` (linha por classe); `investment_profile(id=1)` default
`perfil='custom'`. Idempotente.

## 2. Perfis predefinidos (presets — valores propostos; ajustáveis)
Devolvidos por um endpoint/const. **Somam 100%** cada. Proposta (confirmar com o usuário se quiser outros):

| Classe | Conservador | Moderado | Arrojado |
|---|---|---|---|
| Renda Fixa (`rf`) | 60 | 35 | 10 |
| RF Internacional (`rf_int`) | 10 | 5 | 0 |
| Ações Nacionais (`acoes_nac`) | 10 | 20 | 30 |
| Ações Internacionais (`acoes_int`) | 5 | 15 | 25 |
| ETFs (`etf`) | 0 | 0 | 0 |
| Fundos Imobiliários (`fii`) | 13 | 15 | 15 |
| REITs (`reit`) | 2 | 5 | 10 |
| Criptomoedas (`cripto`) | 0 | 5 | 10 |
| **Total** | **100** | **100** | **100** |

## 3. Endpoints (router `portfolio.py`)
- **`GET /api/investments/targets`** → `{ targets:{classe:pct}, perfil, sum_pct, valid: sum==100 }` (+ labels).
- **`PUT /api/investments/targets`** `{ targets:{classe:pct}, perfil? }` → upsert; **rejeita salvar se a soma ≠ 100**
  (erro `422 {error:{code:'targets_sum', message:'A soma das metas deve ser 100%'}}`). Define `perfil='custom'`
  quando os valores não baterem com nenhum preset, senão grava o nome do preset.
- **`GET /api/investments/profiles`** → os 3 presets (tabela §2).
- (a F4.4 lê `GET /api/investments/targets` para o cálculo do aporte.)

## 4. Frontend — `/investimentos/metas` (azul)
- Header "Metas de Investimento — Edite os itens abaixo para ajustar suas metas." + subtítulo do print.
- **3 cards de perfil** (Conservador/Moderado/Arrojado) — clicar aplica o preset aos sliders (não salva ainda).
  Card selecionado com borda **azul**.
- **Donut central "Total: X%"** (anel azul `invest-ring`) + legenda por classe com o % atual.
- **Sliders 0–100%** por classe (8), com o valor numérico; thumb/track **azul**.
- **Validação/trava (do print):**
  - `Salvar` **habilitado só com soma = 100%** (botão azul; desabilitado/cinza fora disso).
  - **> 100%:** centro do donut em alerta (âmbar→vermelho) com **"Total: X%"** e a mensagem
    **"O valor ultrapassou {X−100}% do valor das metas"**; o arco do excedente "transborda" o anel; micro-animação ao cruzar 100%.
  - **< 100%:** **"Faltam {100−X}% para 100%"** em tom suave.
- Botões **"Resetar valores"** e **"Salvar"** (azul).
- Hooks: `useTargets()` / `useSaveTargets()` (invalida `['investments','targets']` e `['investments','budget']`/aporte);
  `useProfiles()`. Estados loading/erro.

## 5. Edge cases
- Renda Fixa entra normalmente como classe (mesmo sem ticker) — seu % alvo vale para o aporte/rebalance.
- Salvar fora de 100% é bloqueado (UI e API). Mudar um slider recalcula o total ao vivo.
- Trocar de perfil preenche os sliders (preview) mas **só persiste no Salvar**.
