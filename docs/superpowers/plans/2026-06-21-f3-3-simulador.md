> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move the financial simulator into the Next.js app at `/simulador`, preserving the current backend calculations.

**Architecture:** Keep `src/agent/simulator.py` as the source of business logic and add only API characterization tests on the backend. Build a client-side Next route with typed payload/result helpers, React Query mutations, focused UI components, and same-origin `/api/*` calls.

**Tech Stack:** FastAPI, pytest, Next.js App Router, React Query, TypeScript, Vitest, Tailwind, lucide-react.

## Task 3: API Hook

- [ ] **Step 1: Create hook**

Create `web/hooks/useSimulator.ts` with:

```ts
export function useScenarioSimulation() {
  return useMutation({
    mutationFn: (input: ScenarioSimulationRequest) =>
      api.post<ScenarioSimulationResponse>("/simular", input),
  });
}

export function useAmortizationSimulation() {
  return useMutation({
    mutationFn: (input: AmortizationRequest) =>
      api.post<AmortizationResponse>("/amortizacao", input),
  });
}
```

Use the existing `useMaintenance` hook for preloading property data instead of creating another query.
