# F1.2 Tags Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the F1.2 tag workflow: seed the 7 canonical tags, expose CRUD APIs, allow transaction `tag_id` updates, and ship the first functional `/tags` UI plus reusable `TagSelect`.

**Architecture:** Keep the backend aligned with F1.1 buckets: repository functions own validation and database writes, routers translate repository outcomes into HTTP status codes, and app/CLI bootstrap call explicit seed functions. The frontend mirrors the existing bucket hooks and select UX while keeping tag-specific page concerns in `web/components/tags`.

**Tech Stack:** FastAPI, SQLite repository layer, pytest/TestClient, Typer CLI, Next.js App Router, React Query, Vitest, Tailwind, lucide-react.
