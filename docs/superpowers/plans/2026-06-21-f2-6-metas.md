# F2.6 Metas Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build `/metas` with budget bucket planning and savings goals CRUD, keeping Orçamento as the source of truth for planned-vs-spent math.

**Architecture:** Extend `buckets_repo` and `buckets` router for editable planned values and a `GET /api/buckets/plan` endpoint backed by `budget_repo.budget_for_month`. Add a non-destructive `savings_goals` migration plus `savings_repo`/`savings` router that maps DB rows into `summarize_wishlist` instead of reimplementing savings math. The frontend uses TanStack Query hooks and focused components for bucket rows, summary badges, savings cards, and an inline modal.

**Tech Stack:** FastAPI, SQLite migrations/repository layer, pytest, Next.js App Router, React Query, Vitest, Tailwind.
