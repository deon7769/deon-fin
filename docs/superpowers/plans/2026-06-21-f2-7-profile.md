# F2.7 Profile Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the Profile screen and API so name, email, monthly income, financial month start day, and financial goals are editable from the new UI.

**Architecture:** Keep `profile` as the single-row canonical store under `src/web/repositories/profile_repo.py`. The router validates input, updates the singleton row, and schedules reference-month recomputation when the start day changes; the frontend uses React Query hooks and a small form component rather than adding a form library.

**Tech Stack:** FastAPI, SQLite, pytest/TestClient, Next.js App Router, React Query, Vitest, Tailwind, lucide-react.
