# F2.4 Faturas Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `/faturas` workflow with card selection, derived monthly invoice totals, itemized credit-card purchases, inline bucket/tag editing, and category totals.

**Architecture:** Add a thin `invoices` API router backed by `invoices_repo`, reusing `src.agent.cards.CREDIT_TYPES` and `_EXCLUDE` as the source of truth for credit-card purchases. The frontend uses TanStack Query hooks, focused invoice components, and existing `BucketSelect`, `TagSelect`, `DataTable`, `MoneyText`, and `MonthYearPicker` patterns.

**Tech Stack:** FastAPI, SQLite repository layer, pytest, Next.js App Router, React Query, Vitest, Tailwind.
