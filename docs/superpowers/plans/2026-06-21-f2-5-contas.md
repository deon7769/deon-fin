# F2.5 Contas Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the `/contas` workflow with bank/card balances, connected-account actions, account ordering, and Pluggy balance normalization.

**Architecture:** Add account connection metadata and card metadata through additive migrations, then centralize account listing/actions in `accounts_repo` and `accounts` router. The frontend consumes `GET /api/accounts` through TanStack Query and replaces the placeholder with KPI cards, bank/card tables, card view, sync status chips, and basic action buttons.

**Tech Stack:** FastAPI, SQLite migrations/repository layer, pytest, Next.js App Router, React Query, Vitest, Tailwind.

**Execution Mode:** Staged direct execution. Backend and frontend share one API contract, so parallel agents are reserved for validation/review rather than simultaneous edits.
